# 工具栈决策（当前推荐版）

## 推荐的最小可用配置
### 必装
- Obsidian：知识与任务管理
- Cursor：主编码环境
- GitHub Copilot：仓库内快速 agent / PR / 补丁
- 一个高质量对话模型：ChatGPT 或 Claude
- Python：执行确定性流程
- Git / GitHub：版本管理与作品集证明

### 第二阶段再考虑
- OpenClaw：做远程交互和多渠道触发
- 语音输入/播报
- 屏幕监控
- 24小时后台守护

## 为什么不建议你现在就照搬别人那套
你当前最缺的不是“更多常驻 agent”，而是：
- 单一事实来源
- 会话沉淀机制
- repo 级规则
- eval 和 debug 习惯
- 输出资产模板

如果这些没建立好，多开工具只会增加混乱和成本。
