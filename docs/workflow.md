# 工作流总览

## 一、知识层（Obsidian）
- PRD / problem framing
- backlog / sprint
- chat session summary
- memory temp
- decision log

## 二、执行层（AI 工具）
- Cursor：默认编码代理
- Copilot：快速实现 / 本地 agent mode / PR 辅助
- Claude Code：复杂 debug / 架构审查
- DeepSeek：低成本批量结构化
- OpenClaw：可选远程入口，不是必须主控

## 三、确定性层（Python）
- collect
- clean
- normalize
- merge
- export
- eval

## 四、输出层
- research brief
- evidence table
- validated insights
- action matrix
- case deck
- resume bullet
- blog page / B站脚本

## 原则
1. 单项目先跑通，再考虑并行多agent
2. Obsidian 只做“单一事实来源”，不做重复存储
3. 会话记忆分两层：长期规则 / 短期临时上下文
4. 自动化优先替代重复动作，不优先替代判断
