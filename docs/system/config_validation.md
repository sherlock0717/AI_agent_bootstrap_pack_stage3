# Config Validation

## 目的
这套系统里的 YAML 配置文件正在逐渐承担“系统控制平面”的作用。
为了减少后续改动带来的隐藏错误，需要建立配置校验层。

## 当前策略
当前阶段采用两层结构：

### 第一层：Schema 文件
放在 schemas/ 目录中。
作用：
- 明确关键配置文件应该具备哪些字段
- 作为长期结构合同
- 为以后升级到更严格的 schema 引擎做准备

### 第二层：validate_configs.ps1
当前阶段使用低依赖方式做校验，主要检查：
- 关键配置文件是否存在
- 关键字段是否存在
- registry / manifest / local agents 之间的引用是否存在
- 常用路径是否存在
- 标准入口脚本是否存在

## 为什么先不用复杂 YAML 验证器
因为当前阶段优先追求：
- 新手可运行
- 无额外安装负担
- 能尽早发现结构漂移

后续如果系统继续扩展，可以再升级到：
- 更严格的 YAML 解析
- 正式 JSON Schema 校验
- CI 中自动校验
