# Knowledge Playbook

## 目标
知识层用于保存系统长期有效的信息，而不是只靠聊天记录记忆。

## 分类
- system：系统级稳定知识
- projects：项目级知识记录
- sessions：工作会话摘要
- decisions：关键决策记录

## 使用原则
- 能沉淀为长期规则或方法的内容，进入 knowledge
- 一次性执行输出放 outputs
- 运行过程信息放 logs
- 外部项目的业务代码不放入系统母目录

## 建议频率
- 每次重要推进后，至少补一条 session
- 每次关键取舍后，至少补一条 decision
- 每个项目有独立知识文件夹
- 系统级方法变化时，补 system note
