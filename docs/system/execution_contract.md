# 执行层契约

## 目标

让系统在“聊天工具、代码代理、脚本、知识层”之间有稳定边界。

## 工具分工

- ChatGPT / Claude 对话：研究设计、系统设计、总结、审查
- Cursor：改代码、读仓库、重构、补日志
- Copilot：repo 内快速实现与 PR 协作
- Claude Code：复杂 debug 与结构审查
- Python 脚本：所有确定性任务

## 原则

1. 可脚本化的任务不得长期停留在对话里
2. 子项目脚本不得直接修改系统根配置，除非通过 manifest 或 registry
3. 任何新自动化都必须说明：输入、输出、失败处理、日志位置
