# PsyLens Local Rules

## 项目定位
PsyLens 是 AI 系统的试运行与方法验证子项目，用于验证系统在真实项目中的项目接入、规则继承、脚本调用、结果归档和交付流程。

## 项目边界
- PsyLens 不是系统母仓库本体。
- PsyLens 的真实代码仓库位于外部路径，不直接存放在系统母目录中。
- 系统母目录中的 `projects/PsyLens/` 只负责保存项目登记信息、局部规则和接入说明。

## 当前阶段目标
当前阶段优先验证以下内容：
- 外部项目接入是否清晰
- 局部规则是否能与系统总规则配合
- 输出、日志、知识记录是否能归档到统一结构
- 后续是否适合推广为更多项目的接入模板

## 工具优先级
- 代码与脚本修改：优先使用 Cursor
- 复杂调试与规则梳理：可使用 Claude Code
- 系统设计、流程整理、文档规划：可使用 ChatGPT
- 可确定执行的步骤：优先写成脚本，不只停留在对话中

## 输出与归档
- 项目知识记录放在：`knowledge/projects/PsyLens/`
- 项目输出放在：`outputs/projects/PsyLens/`
- 项目日志放在：`logs/projects/PsyLens/`

## 当前限制
当前阶段不把 PsyLens 业务开发作为系统主任务，优先把它作为“系统如何接入真实项目”的验证样板。
