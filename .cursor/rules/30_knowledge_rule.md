# Knowledge Rules

## 适用范围
- knowledge/**

## 目标
这里的内容是系统的长期记忆，而不是临时输出。

## 编写原则
- 记录应可回看、可引用、可复盘
- 尽量保持标题明确
- system / project / session / decision / review / dashboard archive 等类型要分清
- 从 outputs 提升来的内容，应保留来源说明
- 修改知识结构后，记得刷新 index
- knowledge 强调沉淀，不强调运行时细节堆砌

## 不要这样做
- 不要把一次性运行输出直接塞进 knowledge
- 不要把技术日志当成知识笔记
- 不要让 session、decision、review 混成同一种记录
