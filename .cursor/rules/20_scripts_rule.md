# Scripts Rules

## 适用范围
- scripts/**

## 目标
这里的内容优先服务于“可执行、可重复、可验证”的系统动作。

## 编写原则
- 优先写成可以直接运行的脚本，而不是只写描述
- 参数要明确，默认值要合理
- 涉及移动、删除、归档等操作时，优先考虑 DryRun 方案
- 输出要清楚，尽量让新手也能看懂
- 报错要明确，不要静默失败
- 不要把长期知识直接写进脚本输出里，除非有明确提升流程
- 新增标准入口脚本后，考虑是否需要同步到 system_routes.yaml

## 不要这样做
- 不要把业务解释写成大量脚本注释
- 不要把临时路径写死得不可替换
- 不要让脚本既负责执行又负责长期知识归档而没有边界
